## 二、目标 / 非目标

### 目标

- **client 端职责清晰**：虚拟办公室画布（仅 PAIRED 状态显示）+ agent 卡片网格 + 3 类抽屉式弹窗（chat / form / confirm）；零 DB 改动（v6.1 已就绪）；零后端业务逻辑（v6.1 + v6.2 已就绪）；纯前端 UI + lib 包装。
- **5 + 3 + 2 = 10 资源函数封装**：`client/src/lib/agents.ts`（5）+ `sessions.ts`（3）+ `messages.ts`（2），每函数调 `apiFetch` 透传 v6.1 端点。client 端组件**不**直接用 `apiFetch`；统一从 lib/ 包。
- **抽屉式弹窗 + 滑入/滑出动画**：3 类 v6.3 新弹窗（ChatDialog / AgentFormDialog / ConfirmDialog）从屏幕**右侧**滑入（200ms ease-out）/ 滑回右侧（150ms ease-in）+ 透明度 0↔1；overlay 同步淡入淡出。**不**引第三方动画库（`react-transition-group` / `framer-motion` / `radix-ui`），纯 CSS @keyframes + 自定义 hook。
- **统一响应解析**：`apiFetch` 抛 `ApiError(code, message, data)`，v6.3 组件 try/catch 局部处理；不重写错误处理逻辑。
- **i18n 完整集成**：所有新增 UI 文案进 `client/src/i18n/locales/{en,zh-CN}/translation.json`；agent.name / description / systemPrompt **不**进 i18n（用户数据）。修正 v5 现状 409 文案（"无权限访问" → "名称冲突"）以对齐 v6.1 `agent_name_conflict` 业务码。
- **v3 鉴权 + 业务 401 全局处理沿用**：v5 现状 `my-ai:unauthorized` 事件复用；v6.3 组件 catch 到 `ApiError(401)` 派发事件，App 切 NEED_REPAIR + 弹 PairDialog。
- **零网络协议破坏**：v6.3 改的是"加新端点调用 + UI 渲染"；`X-Client-Key` 鉴权、响应 `{data, code, message}` 包装、compat 握手、secure-store schema 全部不变。
- **双层测试**：vitest + @testing-library/react 单元测（hook + lib + 组件）+ 集成测（端到端 office flow）。**不**引 Playwright / tauri-driver / WebDriver；v6.3 阶段与 v5 现状一致。

### 非目标

- **不**做流式 chat 响应：v6.3 同步（v6.1 同步响应不变）；SSE / WebSocket 后续评估。
- **不**做多 chat 弹窗并存：v6.3 一次一个（v6.md §5.5 决策 16 沿用）。
- **不**做 capabilities 编辑 UI：v6.3 form 隐藏 capabilities 字段（决策 26）；agent 卡片只读 chip 展示；用户若需改 capabilities 需走 API 或后续 UI。
- **不**实现其他 LLM provider：v6.3 阶段 form 仍仅 `'openai-compatible'` 可选（v6.1 CHECK 约束）；UI 灰显其他（v6.3 阶段无其他选项）。
- **不**做 session 列表 / 持久化：v6.1 阶段无 `GET /v1/sessions` 列表端点；v6.3 dialog 关闭 = session 引用丢失；点开新 chat dialog = 新 session。
- **不**做 pair 弹窗化：v5 PairBanner / PairDialog / Settings 现状保留；v6.3 决策沿用 v6.md 后续评估。
- **不**做 agent 标签 / 分组 / 搜索 / 软删除 / 启用-停用：v6.3 阶段不做。
- **不**做新 i18n 语种：zh-CN / en 双语按 v5 现状。
- **不**做 E2E 测试：v6.3 与 v5 一致，仅 vitest + RTL；Playwright / tauri-driver 后续 v6.4+ 评估。
- **不**做视觉回归测试：动画靠 CSS @keyframes 写一次，不引 snapshot。
- **不**做 a11y 完整审计：v6.3 仅基础 role / aria-label；后续 v6.4+ 评估。
- **不**做 compat 矩阵版本 bump：v6.3 文档不锁 bump 数字；预留 v6.3 实际写代码时由实施 plan 按 v2 规则（改 client 端 → client bump）定数字。
- **不**复述 v6.1 / v6.2 已锁定的内容：DB schema / 端点契约 / 错误码细化 / 内部 header 注入以 v6.1 §5.x / v6.2 §5.x 为准；v6.3 仅说"调 v6.1 + v6.2 端点"。
- **不**改 v5 现状组件（PairBanner / PairDialog / Settings / MismatchBanner / ThemeToggle / LanguageSwitcher）：v6.3 动画决策明确"仅 v6.3 新弹窗加动画，v5 PairDialog 保持原状"。

**v5 现状 `friendlyApiError`**（在 `PairDialog.tsx`）：

```ts
function friendlyApiError(e: unknown, prefix: string, t: Tx): string {
  if (!(e instanceof ApiError)) return friendlyNetworkError(e, prefix, t);
  const translated = t(`errors.api.${e.code}`); // 走 i18n 资源
  if (translated && translated !== `errors.api.${e.code}`) {
    return `${prefix}：${translated}`;
  }
  // 兜底: 原始 message / "错误码 N"
}
```

**`EmptyOffice.tsx`**（空状态）：

- Props: `onCreate: () => void`

- 无内部状态

**`AgentCard.tsx`**（单个 agent 卡片）：

- Props: `agent: Agent` / `onChat(): void` / `onEdit(): void` / `onDelete(): void`
- 渲染：
  - 卡片主体（除按钮外）click → `onChat()`
  - 右上角浮出按钮（hover 时显示）"✎" → `onEdit()` / "⌫" → `onDelete()`

- 无内部状态；纯展示

**`ChatDialog.tsx`**（agent 弹窗）：

- Props: `agentId: string` / `gatewayUrl: string` / `clientKey: string` / `onClose(): void` / `onAgentDeleted(): void`
- 接入 `useDialogAnimation(onClose)` 拿 `isClosing` / `close` / `onOverlayClick`
- 自管状态：
  - `sessionId: string | null`（初始 null；首条消息发送前生成）
  - `messages: Message[]`
  - `input: string`
  - `sending: boolean`
  - `loadError: string | null`（getAgent 加载失败）
- useReducer 状态机（消息流）：

  ```ts
  type ChatState =
    | { kind: 'idle' }
    | { kind: 'sending'; pending: { tempId: string; content: string } }
    | { kind: 'error'; pending: { tempId: string; content: string } | null; error: string };
  ```

- 消息流：
  1. 打开 dialog → 调 `getAgent(gatewayUrl, clientKey, agentId)` → 404 → 设 `loadError` + 1.5s 后 `onClose()`
  2. 200 → 用 agent.name 渲染顶栏
  3. 用户输入 → 调 `sendMessage(content)`：
     - `input` 清空；按钮 disabled
     - optimistic：push `{ id: <tempId>, role: 'user', content, createdAt: <now> }`（标记 `sending=true`）
     - 若 `sessionId === null` → 调 `createSession(gatewayUrl, clientKey, agentId)` → 存 `sessionId`
     - 调 `postMessage(gatewayUrl, clientKey, sessionId, content)`
     - 成功：替换 tempId 消息为 server `userMessage` + append `assistantMessage`
     - 失败（catch）：标 tempId 消息 `failed=true` + 显示底部红条 + 提供 "重试" 按钮
- agent 被删（`onAgentDeleted()` 触发）：显示半屏提示 1.5s 后 `onClose()`
- 滚动：消息列表底部自动滚到新消息（`useRef + useEffect on messages.length`）
- ESC / overlay click / × 按钮 → 全部走 `useDialogAnimation.close()`（带 150ms 动画）

**`AgentFormDialog.tsx`**（新建/编辑 agent 弹窗）：

- Props: `mode: 'create' | 'edit'` / `agentId?: string`（仅 edit）/ `gatewayUrl` / `clientKey` / `onClose` / `onSaved` / `onDeleted?`（仅 edit）
- 接入 `useDialogAnimation(onClose)`
- 自管 8 字段 form state（`useState`）：

  ```ts
  type FormState = {
    name: string;
    description: string;
    baseUrl: string;
    model: string;
    maxTokens: string; // string 而非 number；空 = 不限
    enabledApi: boolean;
    systemPrompt: string;
  };
  ```

- 自管状态：`submitting: boolean` / `nameConflict: boolean` / `submitError: string | null` / `showDeleteConfirm: boolean`
- 客户端 zod 校验：

  ```ts
  const FormSchema = z.object({
    name: z.string().min(1).max(64),
    description: z.string().max(256).default(''),
    baseUrl: z.string().min(1).max(512),
    model: z.string().min(1).max(128),
    maxTokens: z.union([z.literal(''), z.coerce.number().int().min(1).max(32000)]),
    enabledApi: z.boolean().default(false),
    systemPrompt: z.string().max(8192).default(''),
  });
  ```

- 加载（edit 模式）：打开时调 `getAgent(gatewayUrl, clientKey, agentId)` → 404 → 显示 `agentForm.errors.notFound` + 1.5s 后 `onClose()`
- 提交：调 `createAgent` / `updateAgent`；409 → 设 `nameConflict=true`（name 字段下方红字）；502 / 其他 → 设 `submitError`
- 成功：`onSaved()` → App 关 dialog + refetch
- 删除流程：点 "删除"（仅 edit 模式）→ 设 `showDeleteConfirm=true` → 渲染 `<ConfirmDialog>` → 确认 → 调 `deleteAgent` → `onDeleted()`

**`ConfirmDialog.tsx`**（通用确认弹窗）：

- Props: `title: string` / `message: string` / `confirmLabel: string` / `cancelLabel: string` / `onConfirm(): void` / `onClose(): void`
- 接入 `useDialogAnimation(onClose, { mountEsc: false })` —— 嵌在 AgentFormDialog 内部时不挂 ESC，由父层 AgentFormDialog 统一响应 ESC
- 内部 `useState<boolean> submitting`
- 视觉：固定 420px 宽；竖直居中；走 slideInRight / slideOutRight（与 3 抽屉同款动画语言）
- 提交流程：点 "确认" → `onConfirm()` 由父组件负责实际 API 调用 + 错误处理（不在 ConfirmDialog 内部 catch）

### 5.3 状态机 + 数据流

#### 5.3.1 App.tsx 顶层状态

- **不**用 `useReducer`；officeDialog 简单判别联合 + refetchKey 数字，useState 足够
- Office / ChatDialog / AgentFormDialog / ConfirmDialog 内部各持 `useState` 或 `useReducer`，**不**对外暴露
- App → 子组件的接口全是 props（**不**用 Context）

#### 5.3.2 数据流：发一条消息（ChatDialog 内部）

```
用户输入 "你好" → click 发送
  ↓
dispatch({ type: 'send', content: '你好' })
  ↓
useReducer 切到 sending 态
  ↓
optimistic: messages.push({ id: tempId, role: 'user', content: '你好', createdAt: now })
  ↓
若 sessionId === null:
  POST /v1/sessions { id: new UUID, agentId }
  → session 存到本地
  ↓
POST /v1/sessions/{sid}/messages { id: umId, content: '你好' }
  ↓
[网关 502 / core 不可达 → catch → useReducer 切到 error 态 → 显示错误]
  ↓
成功: 替换 tempId 消息为 userMessage (id = umId) + append assistantMessage
  ↓
useReducer 切回 idle
```

#### 5.3.3 数据流：删除 agent（AgentFormDialog + ConfirmDialog + App 联动）

```
用户点 edit → AgentFormDialog 调 getAgent 加载数据
  ↓
用户点 "删除" 按钮
  ↓
AgentFormDialog 设 showDeleteConfirm=true → 渲染 <ConfirmDialog>
  ↓
用户点 "确认删除"
  ↓
AgentFormDialog 调 deleteAgent(gatewayUrl, clientKey, id)
  ↓
成功: onDeleted() → App:
  · setOfficeDialog(null)
  · setOfficeRefetchKey(k => k + 1)
  ↓
Office useEffect([refetchKey]) 触发 → listAgents() 重拉
```

#### 5.3.4 业务 401 全局处理（v5 现状保留 + 复用）

- v5 现状 `App.tsx` 第 127-134 行已 `window.addEventListener('my-ai:unauthorized', ...)`；触发时 `setStatus('NEED_REPAIR') + setShowDialog(true)`
- v6.3 决策：在 4 个新组件（Office / ChatDialog / AgentFormDialog / ConfirmDialog）的 catch 块中，若 `e instanceof ApiError && e.code === 401` → `window.dispatchEvent(new CustomEvent('my-ai:unauthorized'))`
- 401 之后，**不**调用组件内部的错误显示逻辑（让 App 接管）
- **不**改 v5 现状任何代码

**`client/src/lib/use-dialog-animation.ts`**（新增；3 弹窗复用）：

```ts
// 3 个抽屉式弹窗复用：管理 isClosing 状态 + 可选 ESC 键 + 关闭定时。
// 200ms 打开 / 150ms 关闭（CSS 动画时长；同步 timeout 后才调 onClose 真正卸载）。
// mountEsc 默认 true：3 抽屉都挂 ESC；嵌套场景（ConfirmDialog in AgentFormDialog）
//   → ConfirmDialog 传 { mountEsc: false } 避免双 ESC 监听 + 重复触发 close。
import { useCallback, useEffect, useRef, useState } from 'react';

const CLOSE_MS = 150;

export type UseDialogAnimationOptions = {
  mountEsc?: boolean; // 默认 true
};

export type UseDialogAnimation = {
  isClosing: boolean;
  close: () => void;
  onOverlayClick: () => void;
};

export function useDialogAnimation(
  onClose: () => void,
  options: UseDialogAnimationOptions = {},
): UseDialogAnimation {
  const { mountEsc = true } = options;
  const [isClosing, setIsClosing] = useState(false);
  const timerRef = useRef<number | null>(null);
  const closingRef = useRef(false);

  // 卸载时清理未触发的定时器
  useEffect(
    () => () => {
      if (timerRef.current !== null) {
        clearTimeout(timerRef.current);
        timerRef.current = null;
      }
    },
    [],
  );

  const close = useCallback(() => {
    if (closingRef.current) return; // 二次点击直接忽略
    closingRef.current = true;
    setIsClosing(true);
    timerRef.current = window.setTimeout(() => {
      timerRef.current = null;
      onClose();
    }, CLOSE_MS);
  }, [onClose]);

  // ESC 键监听（嵌套子层可关掉）
  useEffect(() => {
    if (!mountEsc) return;
    const onKey = (e: KeyboardEvent) => {
      if (e.key === 'Escape') close();
    };
    window.addEventListener('keydown', onKey);
    return () => window.removeEventListener('keydown', onKey);
  }, [mountEsc, close]);

  return { isClosing, close, onOverlayClick: close };
}
```

### 5.7 错误处理 + 状态机

#### 5.7.1 客户端错误处理矩阵

| 触发场景                                                    | 错误码      | v6.3 处理                                                           |
| ----------------------------------------------------------- | ----------- | ------------------------------------------------------------------- |
| `name` 重复 (POST/PATCH /v1/agents)                         | 409         | AgentFormDialog：name 字段下方红字 `agentForm.errors.nameConflict`  |
| `sessionId` 不存在 (POST /v1/sessions/{id}/messages)        | 404         | ChatDialog：底部错误 `chat.errors.sendFailed`，1.5s 后关 dialog     |
| `agentId` 不存在 (POST /v1/sessions 或 GET /v1/agents/{id}) | 404         | AgentFormDialog：底部 `agentForm.errors.notFound`，1.5s 后关 dialog |
| OpenAI 兼容服务不可达 (POST /v1/sessions/{id}/messages)     | 502         | ChatDialog：底部 `chat.errors.upstream`                             |
| 网络不通 (fetch reject)                                     | ApiError(0) | ChatDialog：底部 `chat.errors.network`                              |
| 401 鉴权失效                                                | 401         | 派发 `my-ai:unauthorized` 事件 → App 切 NEED_REPAIR + 弹 PairDialog |
| 业务其他 4xx / 5xx                                          | 4xx/5xx     | 通用 `friendlyApiError` 走 `errors.api.<code>`                      |
| 拉取 agents 列表失败 (GET /v1/agents)                       | 任意        | Office 顶部红条 `office.error.loadAgents` + 重试按钮                |

#### 5.7.2 各组件错误状态实现

**Office.tsx**：

```ts
const [state, setState] = useState<{
  agents: Agent[];
  loading: boolean;
  error: string | null;
}>({ agents: [], loading: true, error: null });

useEffect(() => {
  let cancelled = false;
  setState(s => ({ ...s, loading: true, error: null }));
  listAgents(gatewayUrl, clientKey)
    .then(agents => {
      if (!cancelled) setState({ agents, loading: false, error: null });
    })
    .catch((e: unknown) => {
      if (cancelled) return;
      if (e instanceof ApiError && e.code === 401) {
        window.dispatchEvent(new CustomEvent('my-ai:unauthorized'));
        return;
      }
      setState(s => ({ ...s, loading: false, error: friendlyApiError(e, '拉取失败', t) }));
    });
  return () => {
    cancelled = true;
  };
}, [gatewayUrl, clientKey, refetchKey]);
```

**ChatDialog.tsx**：

- useReducer 状态机：'idle' | 'sending' | 'error'
- sending 态：input disabled；显示乐观 user message
- 失败：error 态 + 显示底部红条 + 提供 "重试" 按钮
- 重试：dispatch({ type: 'retry' }) → 重新走 send 流程（不重新创建 session）

**AgentFormDialog.tsx**：

- `submitting: boolean` 状态；submitting 期间所有 input disabled
- 409 → 字段下方红字
- 502 / 其他 → 底部红条
- getAgent 加载失败：404 → 1.5s 后自动关闭 + 提示

**ConfirmDialog.tsx**：

- `submitting: boolean` 状态；删除期间 confirm + cancel 都 disabled
- 失败：底部红条（不自动关闭，让用户重试或取消）

#### 5.7.3 业务 401 全局处理（v5 现状保留 + 复用）

v5 现状 `App.tsx` 第 127-134 行 `useEffect` 监听 `my-ai:unauthorized` 事件；触发时 `setStatus('NEED_REPAIR') + setShowDialog(true)`。**v6.3 在 4 个新组件 catch 块中派发**。
